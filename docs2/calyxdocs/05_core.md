# 05 — calyx-core (identifiers, errors, data model, engine traits)

`calyx-core` is the dependency-light foundation of the workspace: stable identifiers and
content-addressing, the closed `CALYX_*` error catalog, the constellation data model, the
shared enum vocabulary, the engine traits every other crate implements, plus bounded
allocation/cache primitives and a handful of cross-cutting policy types (consent, security,
temporal, cold-start). It has no Calyx dependencies — every other crate depends on it.

## 2. Identifiers and content-addressing (`ids.rs`)

Constant `ID_BYTES = 16`; hex display width `HEX_CHARS = 32`.

### 2.1 ID types

| Type | Underlying | Display form | Generation / format |
|---|---|---|---|
| `VaultId(pub Ulid)` | 128-bit ULID | ULID Crockford-base32 string | Wraps an externally-minted `Ulid`; stable for the life of a vault. `from_ulid`, `as_ulid`. |
| `LensId(pub [u8;16])` | 16 raw bytes | 32-char lowercase hex | Content-addressed (BLAKE3) from lens spec; see 2.2. |
| `CxId(pub [u8;16])` | 16 raw bytes | 32-char lowercase hex | Content-addressed (BLAKE3) from input + panel version + vault salt; see 2.3. |
| `SlotId(pub u16)` | `u16` | decimal | Compact stable panel index. `serde(transparent)`. `new`, `get`, `with_key`. |
| `SlotKey { id: SlotId, key: String }` | — | — | Pairs a `SlotId` with a stable human-readable key. Accessors `id()`, `key()`. |

`LensId` and `CxId` are generated by the `hex_id!` macro, which provides `from_bytes`,
`to_bytes`, `as_bytes`, `Debug` (tuple of lowercase hex), `Display` (lowercase hex),
`FromStr` (via `parse_hex_16`), and serde as a hex **string**. `VaultId` serializes via its
ULID string; `SlotId` is a transparent `u16`. All ID types derive
`Clone, Copy, PartialEq, Eq, PartialOrd, Ord, Hash` (`SlotKey` is not `Copy` and lacks
`Ord`/`PartialOrd`).

### 2.2 `LensId::from_parts`

```
LensId = content_address([ name.as_bytes(), weights_sha256, corpus_hash, output_shape ])
```

Identical lens specs (name + weights hash + corpus hash + output shape) yield the same id
across vaults. Test `lens_id_is_deterministic_on_raw_bytes` confirms determinism and that
changing `output_shape` changes the id.

### 2.3 `CxId::from_input`

```
CxId = content_address([ input_bytes, panel_version.to_be_bytes(), vault_salt ])
```

`panel_version: u32` is encoded big-endian. Determinism and panel-version sensitivity are
pinned by `cx_id_is_deterministic_on_raw_bytes`.

### 2.4 `content_address` — the hash formula

`pub fn content_address<I,P>(parts) -> [u8;16]`. Steps:

1. `blake3::Hasher::new()`.
2. For each part in order: update with `(part.len() as u64).to_be_bytes()` (8-byte big-endian
   length prefix), then update with the part bytes. This **length-delimits** each part so
   `["ab","c"]` and `["a","bc"]` hash differently (`content_address_is_length_delimited`).
3. Take the first 16 bytes of the 32-byte BLAKE3 digest.

So all Calyx content addresses are a **128-bit BLAKE3 prefix over length-delimited ordered
byte parts**.

### 2.5 `ParseIdError`

| Variant | Raised when |
|---|---|
| `InvalidLength { expected, actual }` | hex string length != 32 |
| `InvalidHex { index }` | non-hex byte at the given byte index |
| `InvalidUlid` | `VaultId::from_str` got a non-ULID string |
| `InvalidSlotId` | `SlotId::from_str` got a non-`u16` |

Implements `Display` + `std::error::Error`. Helper `parse_hex_16` accepts both upper- and
lowercase hex; `hex_lower` emits lowercase. Proptests round-trip every ID through
Display↔FromStr and serde-JSON.

## 3. Error taxonomy — the closed `CALYX_*` catalog (`error.rs`)

### 3.1 Root types

- **`CalyxError`** — the error-taxonomy **root type**. Fields:

  | Field | Type | Meaning |
  |---|---|---|
  | `code` | `&'static str` | stable `CALYX_*` wire code |
  | `message` | `String` | concrete failure detail |
  | `remediation` | `&'static str` | stable remediation text |

  Derives `Debug, Clone, PartialEq, Eq, Serialize` and `thiserror::Error` with
  `#[error("{code}: {message}")]`. It is **not `Deserialize`** (the `remediation`/`code` are
  static). Wire JSON shape is exactly
  `{"code":...,"message":...,"remediation":...}` (`error_serializes_to_wire_shape`).

- **`type Result<T> = std::result::Result<T, CalyxError>`** — crate-wide result alias.

- **`CalyxErrorCode`** — closed `enum` of the PRD-18 codes (derives
  `Debug, Clone, Copy, PartialEq, Eq, Hash`). Methods: `code() -> &'static str`,
  `meaning() -> &'static str`, `remediation() -> &'static str`, `error(msg) -> CalyxError`.

- **`CALYX_ERROR_CODES: &[CalyxErrorCode]`** — every catalog code in PRD-18 order.

- **`CalyxWarning`** — non-fatal warning enum, `#[serde(tag="code", rename_all="snake_case")]`,
  one variant `Unprovenanced { surface: String }` (constructor `unprovenanced`). Derives
  `Serialize` **and** `Deserialize`.

The `error_catalog!` macro generates the enum, the `CALYX_ERROR_CODES` slice, the three
`const fn` lookups, an `error(msg)` builder per code, and a snake_case constructor on
`CalyxError` per code (e.g. `CalyxError::lens_dim_mismatch("...")`). `CalyxError::from_code`
attaches the catalog remediation automatically.

## 4. Shared enum vocabulary (`enums.rs`)

All enums derive serde with `#[serde(rename_all = "snake_case")]`; struct-like variants
serialize as `{"variant_snake": {fields}}`. Round-trip bytes are pinned by
`enum_serialization_bytes_are_stable`.

| Enum | Variants | Notes |
|---|---|---|
| `Modality` | `Text, Code, Image, Audio, Video, Protein, Dna, Molecule, Structured, Mixed` | Variant set locked by `modality_variant_set_is_locked` (snake_case wire names). |
| `SlotShape` | `Dense(u32)`, `Sparse(u32)`, `Multi { token_dim: u32 }` | Physical vector shape a lens slot emits. JSON e.g. `{"dense":768}`, `{"multi":{"token_dim":128}}`. |
| `Asymmetry` | `None`, `Dual { a: SlotId, b: SlotId }` | Directional relation for asymmetric slot pairings. |
| `QuantPolicy` | `None`, `TurboQuant { bits_per_channel_x2: u8 }`, `MxFp4`, `Pq { m: u8, nbits: u8 }`, `Float8`, `Binary` | `bits_per_channel_x2 = 7` means 3.5 bpc. `MxFp4` = Blackwell microscaling FP4 (MXFP8 fallback). `turboquant_default()` returns `TurboQuant{bits_per_channel_x2:7}`. |
| `AnchorKind` | `TestPass, TieFormed, Thumbs, Label(String), Reward, SpeakerMatch, StyleHold, Recurrence` | Grounded outcome axis. Derives `Ord` (used as `BTreeMap` key). |
| `SlotState` | `Active, Parked, Retired` | Slot lifecycle. |
| `AbsentReason` | `NotApplicable, Redacted, LensUnavailable, Deferred, LensInactive, Error(String)` | Explicit reason a slot vector is absent. |

## 5. The data model (`model/`)

`model/mod.rs` re-exports all model types and `Ts`. Constellation JSON round-trips are
byte-exact (proptest `constellation_json_roundtrip_is_byte_exact`).

### 5.1 `Constellation` — the atomic record (`model/constellation.rs`)

One input measured by one panel of frozen lenses. Derives `Clone, Debug, PartialEq,
Serialize, Deserialize` (no `Eq` — contains floats).

| Field | Type | Meaning |
|---|---|---|
| `cx_id` | `CxId` | Content-addressed constellation id. |
| `vault_id` | `VaultId` | Owning vault. |
| `panel_version` | `u32` | Panel version used for this measurement (must be > 0). |
| `created_at` | `Ts` (`u64` ms) | Server-stamped creation timestamp. |
| `input_ref` | `InputRef` | Input hash + optional pointer + redaction flag. |
| `modality` | `Modality` | Input modality. |
| `slots` | `BTreeMap<SlotId, SlotVector>` | Per-slot vectors; absent slots are explicit `SlotVector::Absent`. |
| `scalars` | `BTreeMap<String, f64>` | Scalar measurements derived at ingest. |
| `metadata` | `BTreeMap<String, String>` | Verbatim string identifiers / source metadata. `#[serde(default)]`. |
| `anchors` | `Vec<Anchor>` | Grounded outcomes observed for this input. |
| `provenance` | `LedgerRef` | Ledger entry proving input→lens→constellation lineage. |
| `flags` | `CxFlags` | Trust / degradation flags. |

Two metadata contract keys (Leapable Vault integration):
`METADATA_CHUNK_ID = "chunk_id"`, `METADATA_DATABASE_NAME = "database_name"`. Accessors:
`metadata_value(key)`, `chunk_id()`, `database_name()`.

`validate_schema()` enforces: `panel_version != 0`; every slot vector valid (delegates to
`SlotVector::validate_schema`, error prefixed `slot {id}: …`); non-empty scalar keys and
finite scalar values; non-empty metadata keys; every anchor valid. All failures use
`CALYX_RECORD_SCHEMA_VIOLATION`.

### 5.2 `SlotVector` and `SparseEntry` (`model/vector.rs`)

`SlotVector` (`snake_case` tagged enum) — per-slot payload. Derives `Clone, Debug,
PartialEq, Serialize, Deserialize`.

| Variant | Fields | Meaning |
|---|---|---|
| `Dense` | `dim: u32, data: Vec<f32>` | Dense f32 payload. |
| `Sparse` | `dim: u32, entries: Vec<SparseEntry>` | Sparse payload. |
| `Multi` | `token_dim: u32, tokens: Vec<Vec<f32>>` | Multi-vector token payload. |
| `Absent` | `reason: AbsentReason` | Explicit absence — "must never be interpreted as a zero vector". |

`SparseEntry { idx: u32, val: f32 }`. Helpers: `is_absent()`, `as_dense() -> Option<&[f32]>`
(only for `Dense`). `validate_schema()` rules:
- Dense: `dim > 0`, `data.len() == dim`, all finite.
- Sparse: `dim > 0`, each `idx < dim`, no duplicate `idx`, each `val` finite.
- Multi: `token_dim > 0`, ≥1 token, each token length == `token_dim`, all finite.
- Absent: always Ok.

### 5.3 `Anchor` / `AnchorValue` (`model/anchor.rs`)

`Anchor` — a grounded real-outcome observation.

| Field | Type | Meaning |
|---|---|---|
| `kind` | `AnchorKind` | Outcome axis. |
| `value` | `AnchorValue` | Observed value. |
| `source` | `String` | Oracle / human labeler / reward / reality source. |
| `observed_at` | `Ts` | Server-observed timestamp. |
| `confidence` | `f32` | In `[0,1]`; deterministic oracles use `1.0`. |

`AnchorValue` (`snake_case` tagged): `Bool(bool)`, `Enum(String)`, `Number(f64)`,
`OneHot(Vec<String>)`, `Text(String)`, `Vector(Vec<f32>)`.

Validation: `confidence` must be finite and within `[0,1]`; `Number` must be finite;
`Vector` must be non-empty and all finite. Failures use `CALYX_RECORD_SCHEMA_VIOLATION`.

### 5.4 `Slot`, `Panel`, `LensCost`, `SlotResource`, `Placement` (`model/slot.rs`)

`Slot` — a frozen lens slot in a panel.

| Field | Type | Meaning |
|---|---|---|
| `slot_id` | `SlotId` | Compact panel slot id. |
| `slot_key` | `SlotKey` | Human-readable key paired with the id. |
| `lens_id` | `LensId` | Frozen lens content id. |
| `shape` | `SlotShape` | Physical vector shape. |
| `modality` | `Modality` | Measured modality. |
| `asymmetry` | `Asymmetry` | Directional relation. |
| `quant` | `QuantPolicy` | Quantization policy. |
| `resource` | `SlotResource` | Cost + placement; `#[serde(default, skip_serializing_if = "SlotResource::is_default")]`. |
| `axis` | `Option<String>` | Optional semantic axis tag. |
| `retrieval_only` | `bool` | Post-retrieval signal only, not primary recall. `#[serde(default)]`. |
| `excluded_from_dedup` | `bool` | Must not drive dedup. `#[serde(default)]`. |
| `bits_about` | `BTreeMap<AnchorKind, Signal>` | Assay signal per outcome axis; custom `anchor_signal_map` serde (see 5.7). |
| `state` | `SlotState` | Lifecycle state. |
| `added_at_panel_version` | `u32` | Panel version that introduced the slot. |

`Panel`:

| Field | Type | Meaning |
|---|---|---|
| `version` | `u32` | Panel version. |
| `slots` | `Vec<Slot>` | Active or historically interpretable slots. |
| `created_at` | `Ts` | Server-stamped creation time. |
| `kernel_ref` | `Option<LedgerRef>` | Ledger ref for the grounding kernel used with this panel. |
| `guard_ref` | `Option<LedgerRef>` | Ledger ref for the guard calibration used with this panel. |

## 7. Time, clock injection (`time.rs`)

- `type Seq = u64` — monotonic store sequence number.
- `type Ts = u64` — server timestamp in **Unix milliseconds**.
- `trait Clock: Send + Sync { fn now(&self) -> Ts; }` — injected clock; deterministic logic
  must use this rather than `SystemTime::now()`.
- `SystemClock` — wall clock; `now()` = ms since Unix epoch (saturates to `Ts::MAX` on
  overflow; **panics** if before the epoch).
- `FixedClock { ts }` — deterministic clock for tests/FSV; `new(ts)`, `now()` returns `ts`.

## 8. Cosine helper (`cosine.rs`)

- `trait GuardTauProfile { fn tau_for(&self, slot: &SlotId) -> Option<f32>; }` with a blanket
  impl for `BTreeMap<SlotId, f32>`. Lets guard-like policies look up per-slot tau without a
  crate dependency.
- `fn dense_cosine(left: &[f32], right: &[f32]) -> Option<f32>` — fails **closed** (returns
  `None`) when lengths differ, either is empty, any element is non-finite, or the denominator
  is non-positive/non-finite. Otherwise returns `dot / (||l|| · ||r||)` if finite.

## 9. Cold-start trust guard (`cold_start.rs`)

Enforces that a brand-new vault may search but must not label high-stakes answers grounded
until ≥1 real anchor exists.

- `VaultTrustState`: `Provisional` | `Grounded { anchor_count: usize }`.
- `ColdStartGuard { state }`: `new()`/`Default` start `Provisional`; `state()`,
  `anchor_count()`, `record_anchor()` (saturating +1, transitions to `Grounded`),
  `assert_grounded(operation) -> Result<()>` (fails with `CALYX_PROVISIONAL_VAULT` until
  `anchor_count >= 1`), `search_always_ok()` always `true`.

## 10. Consent / purpose gating (`consent.rs`)

- `type Timestamp = Ts` (Unix ms).
- `LawfulBasis` (snake_case, `Display` impl): `Consent, LegitimateInterest,
  ContractPerformance, LegalObligation, VitalInterests, PublicTask`.
- `Purpose` (snake_case): `Search, Intelligence, Reranking, Analytics, Export, AuditOnly`.
- `ConsentTag { lawful_basis: LawfulBasis, permitted_purposes: Vec<Purpose>, expires_at:

## 11. Transport security / authN (`security.rs`)

Lives in core so `calyx-aster` and `calyxd` share it without a cycle (PRD 30 §2). Module-local

## 12. Temporal policy contracts (`temporal.rs`)

Post-retrieval boosting contracts (AP-60: temporal signals stay post-retrieval and never
dominant). Constants: `WEIGHT_SUM_EPSILON = 1e-6`, `DEFAULT_HALF_LIFE_SECS = 3600`,
`DEFAULT_POST_RETRIEVAL_ALPHA = 0.10`, `MAX_POST_RETRIEVAL_ALPHA = 0.10`,
`MAX_CAUSAL_MULTIPLIER = 10.0`, `DEFAULT_RECURRENCE_WEIGHT = 0.05`,
`DEFAULT_MAX_RECURRENCE_BOOST = 0.10`. Six module-local `CALYX_TEMPORAL_*` codes (see 3.3).

| `DecayFunction` | `Linear { max_age_secs: u64 }`, `Exponential { half_life_secs: u64 }`, `Step` | Default `Exponential{half_life_secs:3600}`. |

## 13. Bounded allocation primitives (`alloc/`)

PH56 / Stage S13. Every cap violation surfaces `CALYX_ALLOC_CAP_EXCEEDED` (built by
`alloc_cap_exceeded`, remediation "raise the cap or shrink the working set; allocations fail
closed (A26)"). Constant `CALYX_ALLOC_CAP_EXCEEDED` is `pub`.

`AllocStats { arena_high_water_bytes: usize, arena_resets: u64 }` — metrics snapshot.

### 13.1 `Arena` / `ArenaVec` (`alloc/arena.rs`)

Bump allocator over one pre-allocated, `ARENA_BASE_ALIGN = 4096`-byte-aligned block.

- `Arena::new(cap)` — `cap==0` → `CALYX_ALLOC_CAP_EXCEEDED`.
- `alloc(size, align) -> Result<*mut u8>` — `align` must be a power of two (asserted);
  `size==0` returns an aligned dangling pointer consuming nothing; pads cursor to `align`,
  checked-arithmetic throughout; if padded request would cross `cap`, fails **without**
  advancing the cursor; tracks high-water.
- `alloc_vec::<T>(capacity) -> Result<ArenaVec<T>>` — reserves bytes for `capacity` `T`s
  (ZST/zero-cap returns a dangling vec); borrows the arena for the vec's lifetime.
- `reset()` — O(1) cursor rewind to 0, no `free`, increments `resets`.
- Accessors: `used()`, `capacity()`, `high_water()`, `stats()`.
- `ArenaVec<'a, T>`: `push(value)` fails closed at reserved capacity; `len`, `is_empty`,
  `capacity`, `as_slice`, `as_mut_slice`; `Drop` drops initialized elements in place.

### 13.2 `SlabPool` / `PageAlignedSlabPool` (`alloc/slab.rs`)

Fixed-size object pools with a free list; RAII guards release slots on drop. Exhaustion →
`CALYX_ALLOC_CAP_EXCEEDED`. Constants: `PAGE_SIZE = 4096`, `DEFAULT_EMBED_DIM = 768`,
`VEC_BLOCK_SIZE = 768*4 = 3072`, `ANN_NODE_SIZE = size_of::<AnnNode>()`.

- `AnnNode` `#[repr(C)]`: `id: u64`, `neighbors: [u32; 32]`, `level: u16`, `pad: [u8; 6]`.
- `SlabPool<const SLOT_SIZE: usize>`: `new(cap_slots)` (`0` rejected), `acquire() ->
  SlabGuard`, `cap_slots()`, `held()`, `utilization()`. Slots in `UnsafeCell`; free
  list/state behind `RefCell` so `acquire` needs only `&self`. A double-release
  `debug_assert!`s (panics in debug; refuses the push in release).
- Aliases: `VecBlockPool = SlabPool<3072>`, `AnnNodePool = SlabPool<ANN_NODE_SIZE>`.
- `SlabGuard<'_, SLOT_SIZE>`: `Deref/DerefMut` to `[u8; SLOT_SIZE]`, `slot_index()`.
- `PageAlignedSlabPool`: 4 KiB-aligned for pinned-host CUDA staging. `new(slot_size,
  cap_slots)` — `slot_size` must be a non-zero multiple of `PAGE_SIZE` (**panics**
  otherwise); `cap_slots==0` rejected. `acquire() -> PageSlabGuard`, `slot_size()`,
  `cap_slots()`, `held()`, `utilization()`. Manually `unsafe impl Send` (not `Sync`).
- `PageSlabGuard<'_>`: `slot_index()`, `as_mut_ptr()` (page-aligned), `as_mut_slice()`.

## 14. Bounded LRU+TTL cache (`cache/`)

PH56 / Stage S13. `LruTtlCache<K, V>` — byte-capped, LRU-evicting, per-entry-TTL cache over an
intrusive doubly-linked node arena (O(1) get/insert/evict; no external map crate). Constant
`CALYX_CACHE_EVICTED = "CALYX_CACHE_EVICTED"` is emitted as a `tracing::debug!` event (never a
panic). Jitter RNG is `ChaCha8Rng` seeded with fixed `JITTER_SEED = 0xCA178C0FFEE51D0F` for
reproducibility.

- `InsertResult { evicted: usize }`.
- `new(byte_cap, ttl: Duration, clock: Arc<dyn Clock>)` and `with_jitter(byte_cap, ttl,
  jitter, clock)` — `byte_cap==0` → `CALYX_ALLOC_CAP_EXCEEDED`. TTL/jitter stored as ms.
- `get(&mut, key) -> Option<&V>` — entries past TTL are evicted and counted as a miss; a live
  hit is promoted to MRU.
- `insert(key, value, size_bytes) -> Result<InsertResult>` — a single entry larger than the
  whole cap → `CALYX_ALLOC_CAP_EXCEEDED`; first sweeps expired entries, replaces an existing
  key (not counted as eviction), then evicts LRU tail entries until `used_bytes + size_bytes
  <= byte_cap`.
- `evict_expired() -> usize`; `len`, `is_empty`, `used_bytes`, `byte_cap`, `hit_rate`,
  `evictions`, `expired_total`.
- Expiry = `now + ttl_ms` with `±jitter/2` offset (saturating). The clock is injected
  (`Arc<dyn Clock>`), never `SystemTime::now()` in logic.

## 15. Serialization & determinism summary

- **Format**: serde-driven; tested wire format is JSON (`serde_json`). No `bincode`/`ciborium`
  in this crate. Maps use `BTreeMap` for stable key ordering; many round-trip tests assert
  **byte-exact** re-serialization.
- **IDs**: `LensId`/`CxId` serialize as lowercase hex strings; `VaultId` as a ULID string;
  `SlotId` transparently as a `u16`.
- **Enums**: `#[serde(rename_all = "snake_case")]`; struct variants nest under the snake_case
  tag.
- **Content addresses**: 128-bit BLAKE3 prefix over length-delimited (8-byte BE length +
  bytes) ordered parts.
- **Validating deserializers**: temporal policy types validate on load and fail closed.
- **`CalyxError`**: `Serialize`-only (the `&'static str` code/remediation are not
  round-trippable); wire shape `{"code","message","remediation"}`.

## 16. Gaps / not covered

- `CALYX_TEMPORAL_INVALID_WINDOW` is declared and given a remediation in `temporal_error`,
  but no code path in `calyx-core` raises it (its window type, if any, lives elsewhere).
- `calyx-core` defines the engine **traits** only; concrete `Lens`/`Index`/`VaultStore`/
  `Estimator` implementations live in their owning crates (see the cross-references above).
  Tests here use in-file dummy impls only.
- `SystemClock::now()` panics if the system clock predates the Unix epoch — the only panic on
  a non-misuse path in this crate (alloc-pool misuse such as a non-page-multiple slot size or
  a debug double-release also panics).
- The data-model structs deliberately omit `Eq`/`Hash` where they carry floats
  (`Constellation`, `Signal`, `Slot`, `Panel`, `SlotVector`, `Anchor`, etc.).
- Module-local error codes are intentionally outside the closed PRD-18 catalog; the source
  comments flag this as a pending governance follow-up.
