# Calyx — Technical Documentation

Complete technical reference for **Calyx** (repo `ChrisRoyse/Calyx-Dev`), an
association-native database engine written in Rust. Generated by reading the source code
at `C:\code\Calyx-Dev` synced to `origin/main` commit `2e7b9a2` ("Add kernel-first funnel
search", 2026-06-16). Every statement is derived from code; design intent from the repo's
own planning docs is summarized in doc 23 and clearly distinguished from implementation.

## How this set was produced

Each document lists the **source files it covers** at the top, uses numbered sections,
puts structured data in tables, and traces behavioral claims to specific files. The
authoring rules are in [`_SPEC.md`](_SPEC.md). Where the code is a stub, a `todo!()`, or
diverges from the design plans, each doc says so in a **Gaps / not covered** section.

## Notes on accuracy

- Counts in 22 are static (`git ls-files`/`ripgrep`/`wc`) tallies, not a test-runner
  report; tests were **not executed** while producing these docs.
- Several numbers are pinned-by-test invariants in the code (38 error codes, 30 MCP
  tools, 33 column families) — these are exact, not estimates.
- Auxiliary files in this folder: [`_SPEC.md`](_SPEC.md) (authoring rules) and
  `_open_issues.json` (raw GitHub open-issue dump used to build doc 24).
