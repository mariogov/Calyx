## Goal

Implement `OnnxLens` using `ort` (ONNX Runtime Rust bindings) with the CUDA
EP for portable, embedded-vault use. Load the `.onnx` file from HF cache, run
the embedding session, L2-normalize, and return `SlotVector::Dense`. Must pass
the full frozen contract from PH18.

## Build (checklist of concrete, code-level steps)

- [x] `OnnxLens` struct: `id: LensId`, `model_path: PathBuf`, `dim: u32`,
  `modality: Modality`, `session: ort::Session`.
- [x] `OnnxLens::load(spec: &LensSpec, cache: &HfCacheConfig) -> Result<Self>`:
  - resolve `.onnx` path via `hf_cache::resolve`.
  - compute `sha256` of the file bytes; call `check_weights_sha256`.
  - build `ort::Session` with CUDA EP (`ExecutionProviderDispatch::Cuda(…)`)
    using `error_on_failure`; do not add implicit CPU fallback.
  - expose an explicit CPU-only policy for compatibility FSV and require it to
    print `cpu_explicit,no_cuda`.
  - store `dim = spec.output.dense_dim()`.
- [x] `measure(&self, input: &Input) -> Result<SlotVector>`:
  - tokenize `input.bytes` into `input_ids`, `attention_mask` as
    `ndarray::Array2<i64>`.
  - run `session.run(inputs)` → extract `last_hidden_state` or
    `sentence_embedding` output tensor.
  - mean-pool if needed (last_hidden_state → pool over sequence dim).
  - extract `Vec<f32>`; L2-normalize.
  - call `check_output(vec, spec)`.
  - return `SlotVector::Dense { dim, data }`.
- [x] Tokenizer: reuse `tokenizers` crate with `tokenizer.json` from HF cache.
- [x] `#[cfg(feature = "onnx")]` feature gate (default enabled on aiwonder).
- [x] Integration test `onnx_model_produces_valid_vector` (`#[ignore]`).

## Tests (synthetic, deterministic — known input → known bytes/number)

- [x] unit (CPU EP): load a minimal 2-layer `.onnx` (generated by
  `torch.onnx.export` in test fixtures); measure `b"hello"` → `Dense { dim: 4 }`,
  norm ≈ 1.0, all finite.
- [x] unit: wrong `weights_sha256` → `CALYX_LENS_FROZEN_VIOLATION` at load.
- [x] unit: declared `dim=128` but model outputs 768 → `CALYX_LENS_DIM_MISMATCH`
  at measure time.
- [x] edge (≥3): (1) CUDA EP/kernel unsupported → `CALYX_LENS_UNREACHABLE`,
  no CPU fallback; (2) explicit CPU policy → finite unit-norm vector and
  `cpu_explicit,no_cuda`; (3) invalid UTF-8 or wrong modality →
  `CALYX_LENS_DIM_MISMATCH`.
- [x] fail-closed: session creation or inference failure →
  `CALYX_LENS_UNREACHABLE`.

## FSV (read the bytes on aiwonder — the truth gate)

- **SoT:** integration test output + `.hf-cache/<model>/<model>.onnx` file on
  aiwonder
- **Readback:** `cargo test -p calyx-registry onnx -- --include-ignored --nocapture 2>&1`
- **Prove:** output shows explicit CPU readback `dim=384 norm=1.000±0.001`
  with `cpu_explicit,no_cuda`; default CUDA readback prints
  `cuda:0,error_on_failure,no_cpu_fallback` and either a valid CUDA vector or a
  fail-loud `CALYX_LENS_UNREACHABLE`; `ls`/`sha256sum` of the `.onnx` file is
  attached to the GitHub issue.
